本地部署hexo

hexo init 初始化

在你想保存博客内容的路径（文件夹）下里右键 git bash
得到目录结构

  • _config.yml是整个博客的全局配置
  • source是存放生成博客内容所需资源的地方
  • source/_posts是文章的默认存放处
  • themes是博客主题的存放点
  • scaffolds是模板的存放点，这里的模板指的是新建文章的初始内容，Hexo已经默认在此文件夹生成了post.md page.md draft.md
  • npm install
    hexo运行的各种插件
  • hexo generate（重新生成博客内容）
  • 博客内容有改动
  • 多出public文件夹- - - - 博客内容
  • hexo server ，打开 http://localhost:4000
    • 打不开
    • 默认端口 4000，改成8888
    • hexo server - p 8888
    • 打开http://localhost:8888

设置git 推送

下载插件（让hexo能用git推送）
npm install hexo- deployer- git - - save

主目录_config.yml

  • 修改

    deploy:
    type: git
    repo: git@github.com:apaat- kcalm/apaat- kcalm.github.io.git
    branch: master

发布

  • hexo deploy
  • 点开 你的用户名.github.io

常用命令

hexo g = hexo generate 生成博客内容
hexo s = hexo server 本地预览
hexo d = hexo deploy 远程推送

正文

  • hexo clean 清楚已生成的旧文件 hexo n “文章标题” 新建————编辑（.md）

草稿

  • hexo new draft “new draft title”
  • 发布：hexo publish draft “your draft title”

搜狗短语

  • hexo s - g 先hexo g,再hexo s
